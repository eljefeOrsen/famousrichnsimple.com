Infrastructure & cloud usage

FamousRichNSimple is intentionally multi-cloud. Google Cloud hosts our orchestration and AI reasoning layer; AWS runs bursty media workers and storage. This separation lets us keep costs predictable while experimenting with the latest AI models and media tooling.

Google Cloud (control plane)

  • Run Orsen AI APIs on Cloud Run.
  • Store configuration, prompts and templates in Cloud Storage.
  • Leverage Vertex AI / partner LLM APIs for multilingual generation.

AWS (worker plane)

  • Containerized media workers on Amazon ECS.
  • Queued jobs via Amazon SQS.
  • Raw and rendered assets in Amazon S3.
